## 从一颗"脏"豆子说起：Dry Bean Dataset 全流程机器学习实战纪实

### 一、拿到数据：13,611 颗豆子和它们的 16 张"体检报告"

这份数据集来自土耳其科贾埃利大学，目标是通过 16 项几何与形状特征来自动识别 7 个品种的干豆：BARBUNYA、BOMBAY、CALI、DERMASON、HOROZ、SEKER、SIRA。这在农业自动化分拣领域有直接的应用价值——想象一条传送带上成千上万颗豆子飞速滚过，靠人工逐一甄别品种显然不现实，而一台搭载了分类模型的摄像头却可以在毫秒之间完成判定。

每颗豆子的"体检报告"包含 16 个数值特征，涵盖了面积（Area）、周长（Perimeter）、长轴长度（MajorAxisLength）、短轴长度（MinorAxisLength）、宽高比（AspectRation）、偏心率（Eccentricity）、凸包面积（ConvexArea）、等效直径（EquivDiameter）、伸展度（Extent）、密实度（Solidity）、圆度（roundness）、紧凑度（Compactness）以及四个形状因子（ShapeFactor1~4）。

教师事先将全部 13,611 个样本划分成了三份：训练集 9,527 个、验证集 1,347 个、测试集 2,737 个。训练集用来让模型"学习"，验证集用来调参和监控过拟合，测试集则完全保留到最后，作为一场"闭卷考试"来检验模型的真实泛化能力。

从类别分布上看，7 个品种的占比并不均匀：DERMASON 数量最多，在训练集中占 26.3%（2,503 个）；BOMBAY 最少，仅占 3.8%（361 个）。第二多的 SIRA 占 19.3%（1,837 个），紧随其后的 SEKER 占 14.8%，HOROZ 占 14.1%，CALI 占 12.1%，BARBUNYA 占 9.7%。这种类别不平衡意味着模型在面对少数类（尤其是 BOMBAY）时容易出现"偏向多数类"的倾向，后续评估时需要同时关注 Precision、Recall 和 F1 而非只看 Accuracy。

### 二、数据污染：一份名副其实的"脏"数据集

这份数据集的全名叫做 "Dry_Bean_Dataset_**Dirty**"，"Dirty" 三个字绝非虚言。在打开 CSV 文件后，我们发现了四类典型的数据污染：

**第一类污染：标签文本损坏。** 这是最隐蔽也最有趣的一种。原本应该只有 7 种取值的品牌标签（Class 列），在原始数据中实际出现了 **25 种不同的文本变体**。损坏方式有三种：一是"火星文"替换——BOMBAY 被写成 "B0MBAY"（字母 O 被替换为数字 0）、DERMASON 被写成 "D3RMAS0N"（E→3、O→0）、HOROZ 变成了 "H0R0Z"、SEKER 变成了 "S3K3R"；二是尾部多余空格——如 "BARBUNYA "、"DERMASON "；三是大小写混乱——如 "barbunya"、"cali"。仅训练集中就有 406 个样本的标签被污染，验证集 61 个，测试集 114 个，三个集合合计 **581 条脏标签**。如果不做处理，LabelEncoder 会将每种变体都视为独立类别，把 7 类分类任务变成 25 类，其中像 "BOMBAY "（仅 1 个样本）这样的类别对模型来说几乎不可能学会。

**第二类污染：数值列中的文本混入。** 部分数值特征列中混入了非数字内容，例如 Solidity 列有 202 个单元格被 "?" 占位符填充（训练集），还有部分单元格的数值后面附带了单位文本（如 "0.9293 cm"），导致整列被 pandas 识别为字符串类型而非数值类型。

**第三类污染：缺失值。** 训练集中 Perimeter（周长）有 469 处缺失（4.92%），Solidity（密实度）有 272 处 NaN 缺失（2.86%）外加 202 处 "?" 占位符（2.12%），合计涉及 943 条数据。验证集和测试集也有类似比例的缺失。

**第四类污染：极端离群值与异常录入。** 用 IQR（四分位距）法扫描清洗后的训练集 16 个数值列，一共检出了 **5,008 个离群点**。其中 Solidity 有 611 个、Eccentricity 有 592 个、ShapeFactor4 有 529 个。此外，还发现了 Area 列存在负值（面积不可能为负，明显是录入时多打了一个负号）。

### 三、数据清洗与特征工程：一套五级处理流水线

针对上述四类污染，数据管道（`core/dataloader.py`）设计了一套严谨的五级处理流水线：

**第一级：标签归一化。** 对 Class 列依次执行去除首尾空格（`.str.strip()`）、统一转大写（`.str.upper()`）、以及"火星文"纠错映射（D3RMAS0N→DERMASON、S3K3R→SEKER、B0MBAY→BOMBAY、H0R0Z→HOROZ）。这一步将 25 种变体干净利落地归并为 7 个标准类别。

**第二级：数值列文本清洗。** 对所有被识别为字符串类型的特征列，用正则表达式 `[^\d\.\-]` 剔除数字、小数点和负号以外的所有字符（包括 "cm"、"?" 等），然后转换为数值类型。无法转换的一律标记为 NaN。

**第三级：数学公式恢复缺失值。** 这是整个数据管道中最精妙的一步。我们没有简单粗暴地用中位数填充所有缺失值，而是利用干豆几何特征之间的物理关系进行精确的数学重建：Solidity（密实度）的定义就是 Area / ConvexArea，因此当 Solidity 缺失但 Area 和 ConvexArea 都完好时，直接用公式计算还原；Perimeter（周长）与 Area（面积）和 roundness（圆度）之间存在确定的数学关系（$P = \sqrt{4\pi A / R}$），因此 Perimeter 的缺失值同样可以精确恢复。这一步的效果是：训练集 474 个 Solidity 缺失和 469 个 Perimeter 缺失被 **100% 精确恢复**，剩余 NaN 数量为 0。验证集和测试集同样做到了零残留。与中位数填充相比，数学恢复保留了数据的真实分布特征，不会人为引入"所有缺失豆子都具有中等周长"这种不合理的假设。

**第四级：异常值处理。** 首先对 Area 列取绝对值，修正负面积的明显录入错误。然后对每个数值特征计算均值和标准差，将超出 [均值 - 3倍标准差, 均值 + 3倍标准差] 范围的值强制"裁剪"到边界（3-Sigma 截断）。这一步的思想来自统计学中的经验法则：在正态分布下 99.7% 的数据都落在 3 倍标准差之内，超出这个范围的值大概率是噪声或录入错误。截断而非删除的好处是不会损失样本量。

**第五级：标准化与 PCA 降维。** StandardScaler 将所有特征缩放到零均值、单位方差，让 SVM 的核距离计算和 ANN 的梯度下降不再被量纲差异干扰。随后执行 PCA 主成分分析——我们计算了 16 个特征之间的相关系数矩阵，发现了大量高度冗余的特征对：Area 和 ConvexArea 的相关系数高达 0.9999（几乎是同一个信息的复制品），Compactness 和 ShapeFactor3 的相关系数为 0.9987，AspectRation 和 Compactness 为 0.9878。PCA 结果显示，第一个主成分独自解释了 51.35% 的方差，前两个累计 77.44%，到第六个主成分（PC6）时累计方差贡献率达到 **97.45%**，超过了 95% 的经验阈值。项目中默认选用 10 个主成分（累计贡献率 99.95%），在几乎不损失信息的前提下将特征维度从 16 降到了 10，同时彻底消除了特征间的相关性冗余。

### 四、三种分类算法：各有各的"聪明方式"

在清洗降维后的数据上，我们实现了三种截然不同的多分类算法，每种都用自己独特的方式来学习区分豆子的品种：

**支持向量机（SVM）** 是一种经典的几何学派方法。我们使用了 RBF 核函数（径向基函数），它的核心思想是把低维空间中线性不可分的数据"弹射"到一个更高维的空间里，在高维空间中寻找一个最优超平面来分割不同类别的豆子。SVM 的"聪明"之处在于它只关心那些离决策边界最近的少数样本（称为"支持向量"），因此天然具有一定的抗过拟合能力。最终选用的超参数为 C=10.0（控制误分类的惩罚力度）配合 RBF 核。

**随机森林（Random Forest）** 走的是"群众投票"路线。它一次性种下 150 棵决策树，每棵树只看一部分样本和一部分特征，各自独立做出分类判断，最后以多数投票决定最终类别。这种集成学习策略（Bagging）使得随机森林对噪声和离群点有极强的免疫力——即使某些树被噪声误导了，其他树也能"投对"来纠偏。我们将最大分裂深度（max_depth）限制在 12，防止单棵树长得太深太细而记住训练数据的噪声。

**人工神经网络（ANN）** 是三种算法中结构最复杂的。我们搭建了一个三层前馈神经网络：输入层（10 维 PCA 特征）→ 第一隐藏层（128 个神经元，配 BatchNorm + ReLU + Dropout）→ 第二隐藏层（64 个神经元，同样配置）→ 第三隐藏层（32 个神经元，BatchNorm + ReLU）→ 输出层（7 个类别的 Softmax 概率）。

这个架构中融合了多个课堂核心知识点：ReLU 激活函数通过引入非线性让网络能够学习复杂的决策边界（纯线性层无论叠加多少层都等价于一个线性变换）；BatchNorm 对每个 mini-batch 的隐藏层输出做归一化，平滑了损失函数的"地形"，让优化器更快找到谷底；Dropout 在训练时随机关闭 30% 的神经元，强迫网络不能依赖任何单个神经元，起到了正则化防过拟合的效果；Kaiming 初始化确保初始权重的方差与神经元数量匹配，防止梯度在层层传播中消失或爆炸。

此外，网络输出端引入了 Softmax 置信度过滤机制：对每个预测样本，如果模型的最大输出概率低于 0.4，系统会将该预测标记为"拒绝判定"（-1），相当于告诉操作员"这颗豆子我不确定，请人工复检"。这在工业级高速分拣场景中有重要的实际意义——宁可多剔除几颗让工人复检，也不要把 A 品种错判成 B 品种混入包装。

训练阶段，网络最多跑 100 个 epoch，配合 ReduceLROnPlateau 学习率调度器（当验证集 Loss 连续 5 个 epoch 不下降时将学习率减半）和 Early Stopping（验证集 Loss 连续 12 个 epoch 不下降时提前终止训练并恢复到历史最优权重）。实际运行中，网络在第 56 个 epoch 触发了早停，从 0.91 的初始训练 Loss 收敛到了 0.23，验证 Loss 从 0.38 收敛到了 0.19。

### 五、实验结果：三个模型在闭卷考试中的表现

在 2,737 个测试样本的闭卷考试中，三个模型交出了如下成绩单：

SVM 以 **93.56%** 的准确率拔得头筹，Precision 为 93.58%，F1 为 93.57%。它在训练集上的准确率为 93.73%，与测试集仅差 0.16 个百分点，几乎没有过拟合。从混淆矩阵来看，SVM 对 BOMBAY 的识别堪称完美（111 个样本 100% 正确，因为 BOMBAY 独特的短圆形状极易区分），SEKER 识别率 95.9%，HOROZ 95.1%，CALI 94.8%，BARBUNYA 93.7%，DERMASON 93.1%。SIRA 的识别率相对最低（89.0%），主要的误判集中在 SIRA 被错判为 DERMASON（39 例）——从几何特征上看，这两个品种在面积和周长上存在较大的重叠区间。SVM 的推理速度为每秒 14,111 个样本（约 0.07 毫秒/样本），对于实时分拣场景绰绰有余。

ANN 以 **93.53%** 的准确率紧随其后，与 SVM 的差距仅有 0.03 个百分点。但 ANN 的过拟合表现堪称完美——训练集准确率 93.51% 甚至略低于测试集的 93.53%，差值为 -0.02%，这说明 Dropout + 早停 + BatchNorm 的三重正则化组合拳把过拟合压制到了几乎不存在的程度。在 2,737 个测试样本中，ANN 仅"拒绝判定"了 1 个（拒绝率 0.04%），而在剩下的 2,736 个置信样本上准确率高达 93.56%。ANN 的推理速度极为惊人：每秒 398 万个样本（0.0003 毫秒/样本），这得益于 PyTorch 的矩阵运算优化。

随机森林的准确率为 **92.50%**，略低于前两者。它的训练集准确率为 98.55%，过拟合差值为 6.05%，虽然已通过限制 max_depth=12 大幅改善，但仍是三者中过拟合最明显的——这是因为随机森林的每一棵树都有"记住"训练数据的倾向，即使限制了树深也难以完全消除。不过，RF 有一个独到的优势——抗噪鲁棒性（后文详述）。推理速度方面，RF 达到每秒 70,946 个样本，居中水平。

### 六、噪声压力测试：把考试环境"搞恶劣"

一个模型在干净数据上表现好还不够，真正考验实力的是在恶劣环境下的稳定性。我们从三个维度对模型施加了渐进式噪声干扰，观察精度衰减曲线：

**高斯噪声（模拟传感器漂移）：** 在特征值上叠加均值为 0、标准差从 0.01 递增到 0.5 的高斯随机扰动。这是最"温和"的一种噪声——当标准差为 0.1 时，SVM 的精度仅从 93.56% 降到 93.27%，ANN 从 93.53% 降到 92.64%，RF 从 92.50% 降到 91.36%，三个模型都保持了相当的战斗力。即使标准差拉大到 0.5（极其恶劣的工业环境），SVM 仍保持 86.28%，RF 为 80.99%，ANN 为 76.39%。SVM 在高斯噪声下表现出了最强的稳定性。

**椒盐噪声（模拟数据位翻转）：** 随机将一定比例的特征值替换为极大值（"盐"）或极小值（"椒"）。这种噪声的破坏力远超高斯噪声——当噪点比例达到 10% 时，SVM 的精度已经暴跌到 46.12%，ANN 为 53.37%，而 RF 仍然顽强地维持在 **82.46%**。在 30% 的极端椒盐噪声下，RF 以 65.28% 的精度遥遥领先于 ANN 的 28.28% 和 SVM 的 20.56%。这完美地展示了集成学习的鲁棒性优势：即使某些决策树被椒盐噪声带偏了，其他树仍然能通过多数投票"纠偏"回来。

**特征屏蔽（模拟特征丢失）：** 随机将一定比例的特征值直接置零，模拟某些传感器失效或被遮挡的情况。在 50% 特征被屏蔽的极端条件下，ANN 以 55.57% 略微领先（SVM 为 50.64%，RF 为 55.17%），三者表现接近——这说明 PCA 降维后的 10 个主成分中，每个都承载了较为均匀的判别信息，没有哪个主成分是"废特征"。

### 七、过拟合深度分析

过拟合对比图直观地呈现了三个模型的不同"性格"。SVM 的训练集精度 93.73% 与测试集 93.56% 几乎完全重合（差值仅 0.16%），说明 RBF 核 SVM 在 C=10.0 的配置下找到了一个完美的泛化平衡点。ANN 甚至出现了测试集精度（93.53%）略高于训练集（93.51%）的"反直觉"现象——这并非过拟合的反面，而是 Dropout 在训练时人为" handicap "了网络的表达力，导致训练 Loss 偏高；验证和测试时 Dropout 关闭，网络以完整参数运行，表现反而更好。这一现象从 Loss 曲线中也能得到印证：训练 Loss（0.23）始终高于验证 Loss（0.19），两者同步下降且没有发散趋势，证明正则化起到了极佳的效果。

RF 的情况则不同：训练集 98.55% vs 测试集 92.50%，6.05% 的差距虽然在 max_depth 限制下已大幅收窄，但仍说明集成学习中的每棵树存在"记住"训练样本的倾向。在论文中，这可以作为一个很好的分析素材——探讨如何通过进一步降低 max_depth 或增加 min_samples_split 来换取更好的泛化。

### 八、工程总结

整个项目最终集成为一个模块化的命令行工程系统：数据加载与五级清洗模块（dataloader.py）、三模型定义与训练模块（models.py）、多维评估模块（evaluator.py）、深度 EDA 脚本（eda.py）和数据驱动的动态绘图脚本（plot_results.py）。通过一条命令 `python main.py --model all --add_noise --pca_components 10`，系统会自动完成数据加载 → 标签归一化 → 文本清洗 → 数学恢复缺失值 → 3-Sigma 截断 → 标准化 → PCA 降维 → 三模型训练 → 全维度评估 → 结果 JSON 序列化 → 对比报告生成 → 12 张可视化图表绘制 → 论文大纲框架输出的完整端到端流水线。所有实验图表均从 results.json 动态读取，重跑实验后图表自动更新，杜绝了硬编码数据的风险。
